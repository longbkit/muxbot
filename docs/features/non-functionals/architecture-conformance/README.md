# Architecture Conformance

## Purpose

Use this folder for cross-cutting work that keeps implementation aligned with the repository architecture documents.

## Typical Topics

- route and API boundary review
- model taxonomy cleanup
- ownership and naming standardization
- removal of duplicate mutation or serialization paths

## Workflow

- track area state in `docs/features/feature-tables.md`
- keep recurring audit evidence in `docs/audits/architecture-conformance/`
- track executable work in `docs/tasks/`
- keep this folder focused on overview, risks, constraints, and links
