# Release Notes

## Purpose

Use `docs/releases/` as the canonical home for `clisbot` release notes.

This folder is the long-form source of truth for what changed, why it matters, and what an operator or end user should notice.

GitHub Releases should stay shorter and link back here.

## File Layout

- [`unreleased.md`](unreleased.md): draft notes for work that has landed but is not yet grouped into the next published note
- `vX.Y.Z.md`: one file per shipped version

## Writing Rule

Release notes here should optimize for:

- very easy to scan
- clear user impact
- grouped changes instead of commit-by-commit noise
- plain language first, technical detail second
