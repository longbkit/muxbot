# clisbot v0.1.45 Operator Brief

Date: 2026-04-28

## Read This If

- You run a `0.1.43` clisbot install and plan to update.
- You use Slack or Telegram groups, Telegram topics, loops, or streaming previews.
- You rely on clisbot agents to change routes, auth, schedules, or runtime config from chat.

## TL;DR

This release makes clisbot safer for real group and topic usage: shared-surface access is clearer, config update is automatic, scheduled loops are more reliable, prompt context is more readable, and streaming previews are stricter about staying inside the right session.

## What Changed

- Shared group access is easier to reason about: multi-user surface admission and sender policy are now separate.
- Configs from `0.1.43` are backed up and updated automatically to schema `0.1.45`.
- Wall-clock loops now have stronger timezone handling and better first-prompt reliability.
- Prompts now include compact sender and surface context across normal, queue, steering, and loop paths.
- Telegram voice/audio files now reach agent workspaces as attachments.
- Telegram topics and streaming previews have stronger session isolation so old pane content and wrong-topic updates are not rendered as the current answer.

## What To Use Now

- Use `topic:<chatId>:<topicId>` for Telegram topic routes.
- Use `group:*` as the default shared sender policy node for one bot.
- Use exact group or topic routes without route-local `policy` when they should inherit from `group:*`.
- Use `clisbot timezone get|set|clear|doctor` for app-level timezone control.
- Ask agents to run `clisbot auth get-permissions --sender <principal> --agent <id> --json` before sensitive config changes.

## What To Watch

- Existing exact shared routes should not unexpectedly carry `policy: disabled`.
- Calendar loops freeze their resolved timezone when created; changing app timezone later should not move existing loops.
- Slack display enrichment needs the lookup scopes documented in the Slack manifest guide.
- Streaming fixes are intentionally strict about prompt boundaries; report any case where old pane content or another topic's output appears in a new answer.

## Update / Rollout

- Install or run the `0.1.45` package.
- Restart clisbot and let the automatic config update run.
- Confirm `clisbot status` reports schema/runtime health.
- Check Telegram topic routes and any Slack shared routes that matter.
- Keep the backup under `~/.clisbot/backups/` until the updated runtime is stable.
