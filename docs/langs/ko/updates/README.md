# 업데이트 가이드

## 목적

`docs/updates/`는 사용자와 운영자가 다음을 빠르게 파악하도록 돕는 문서 계층입니다.

- 지금 무엇이 중요한가
- 업데이트 후 무엇부터 써야 하는가
- 올바른 업그레이드 경로는 무엇인가
- 운영 리스크가 어떻게 달라졌는가

## 핵심 문서

- [update-guide.md](./update-guide.md)
- [v0.1.50 릴리스 가이드](./releases/v0.1.50-release-guide.md)

## 언제 읽어야 하나

- `clisbot` 업데이트 전
- 최근 변경을 짧게 파악하고 싶을 때
- 릴리스가 setup 또는 ops 방식에 영향을 주는지 판단할 때

## 기본 원칙

- 버전 사실의 기준은 `docs/releases/`
- manual action 판단은 `docs/migrations/index.md`부터
- `updates` 문서는 짧고, 분명하고, 바로 행동할 수 있어야 함
