# clisbot v0.1.39

Released: 2026-04-18

이 릴리스는 현재 `clisbot` 형태에 대한 첫 번째 큰 release note입니다.

## 한 줄 요약

`clisbot`은 이제 얇은 CLI bridge라기보다 실제 Slack / Telegram assistant에 훨씬 가까워졌습니다. 시작이 쉬워졌고, native channel surface 안에서 읽기 쉬워졌으며, 노출이 더 안전해졌고, 긴 작업도 더 믿을 수 있게 되었으며, 일상 사용성이 좋아졌습니다.

## 체감되는 변화

### 1. 첫 bot 시작이 훨씬 쉬워짐

- `clisbot start --cli ... --bot-type ...` 경로가 훨씬 직접적
- Slack / Telegram setup guide가 훨씬 좋아짐
- persistent credential과 runtime-only credential을 이해하기 쉬워져 시험 운용이 빨라짐
- `CLISBOT_HOME`으로 dev / test / second bot 환경 분리가 쉬워짐

### 2. 채팅 안에서 bot이 더 native하게 느껴짐

- Slack / Telegram을 단순 text output이 아니라 실제 product surface처럼 다룸
- rendering이 좋아져 읽기 쉬워짐
- routing, thread / topic continuity, surface-specific reply behavior가 예측 가능해짐
- recent conversation replay로 active conversation context를 더 자연스럽게 이어 받음

### 3. 긴 작업을 더 믿을 수 있게 됨

- streaming preview, running indicator, final settlement가 더 truthful
- tmux startup / submit / attach / detach / recovery 흐름이 크게 강화됨
- 느린 시작이나 runner hiccup, mid-run interruption에도 덜 깨짐

### 4. 반복 작업 자동화 가능

- `/loop` 도입
- loop inspect / cancel CLI도 추가
- queue / loop start notification으로 background work 가시성 향상
- `/nudge`로 멈춘 run을 가볍게 밀어 줄 수 있음

### 5. 공유 사용이 더 안전하고 명확해짐

- DM 기본값이 open이 아니라 pairing-oriented access
- app auth / routed auth가 강화됨
- first-owner auto-claim으로 첫 secure bootstrap이 덜 어색함
- Slack shared-channel 기본값이 더 안전함

### 6. 운영자가 더 많이 보고 디버깅할 수 있음

- status와 runtime summary가 더 풍부하고 정직해짐
- `runner watch --latest` 등 runner debug surface가 강화됨
- setup / workflow / operator docs가 bot-first mental model로 재정리됨

### 7. runner 선택지가 넓어짐

- Gemini CLI runner 지원이 들어옴

## 중요한 업그레이드 메모

- `v0.1.39`는 공식 config shape와 주요 CLI surface에 breaking change가 있습니다
- 이 버전 이전의 설치본이 있다면, 업그레이드 전에 repo 안의 Codex나 Claude에게 현재 config를 `v0.1.39` 형태로 바꾸게 하는 것이 안전합니다

## 업그레이드 경로

```bash
clisbot stop
npm install -g clisbot
clisbot start
clisbot --version
```
