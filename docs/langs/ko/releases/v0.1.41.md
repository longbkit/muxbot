# clisbot v0.1.41

Released: 2026-04-19

`v0.1.41`은 operator-facing 동작을 더 truthful하고 이해하기 쉽게 만듭니다.

## 한 줄 요약

`clisbot`은 이제 DM에서 admission과 behavior를 더 분명히 분리하고, operator가 Slack thread, Slack DM, Telegram topic을 CLI에서 어떻게 지정해야 하는지도 훨씬 명확해졌습니다.

## 핵심 변화

### 1. DM 동작이 더 안전하고 덜 애매해짐

- `dm:*`가 DM admission을 맡음
- exact DM route는 behavior override에 집중
- pairing approval은 이제 `channel + botId` 범위로 기록
- 잘못된 exact DM auth mutation은 route CLI guardrail이 거부

### 2. Loop CLI targeting이 더 명시적

- `clisbot loops`가 Slack channel/group, Slack DM, 기존 Slack thread, 새 Slack thread를 하나의 addressing path로 지원
- Telegram은 `--thread-id` 대신 `--topic-id`
- Slack은 `--new-thread` 지원

### 3. message-tool reply guidance 개선

- agent reply prompt가 Telegram용으로 `--topic-id`를 보여 줌
- `message send`, `message poll` help가 Slack / Telegram 차이를 명시

### 4. CLI runner 관련 operator guidance 강화

- `README.md`에 Codex / Claude / Gemini용 operator note 추가
- 각 CLI용 user guide 추가

## 업그레이드 노트

- exact DM route에 admission 관련 값(`policy`, `allowUsers`, `blockUsers`)을 넣어 쓰고 있었다면 `dm:*`로 옮기세요
- Telegram CLI reply workflow에서 `--thread-id`를 썼다면 `--topic-id`로 바꾸세요

## Validation

- `bun run check`
