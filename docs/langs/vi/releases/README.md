# Ghi chú phát hành

## Mục đích

`docs/releases/` là lịch sử version chuẩn của `clisbot`.

Nhóm này trả lời:

- version đó đổi gì
- tác động với người dùng và người vận hành là gì
- những thông tin nào quan trọng cho update và validation về sau

## Nên kết hợp với gì

- [Hướng dẫn update chi tiết](../updates/update-guide.md)
- [Tổng quan updates](../updates/README.md)
- [Migration index](../../../migrations/index.md)

## Cấu trúc file

- [upcoming.md](./upcoming.md): vùng gom nội dung cho release note kế tiếp
- `vX.Y.Z.md`: mỗi version một file
- [templates/release-note.md](./templates/release-note.md): template chuẩn

## Cách đọc nhanh

- Muốn biết “phiên bản mới nhất đáng chú ý gì”: đọc `v0.1.45`
- Muốn theo dõi các thay đổi sắp public: đọc `upcoming.md`
- Muốn đọc bản tóm tắt ngắn hơn: sang [tổng quan `updates`](../updates/README.md)
