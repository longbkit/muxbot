# Architecture Conformance

## Mục đích

Dùng folder này cho công việc cắt ngang giúp implementation bám theo các tài liệu kiến trúc của repository.

## Chủ đề điển hình

- review boundary của route và API
- dọn model taxonomy
- chuẩn hóa ownership và naming
- loại bỏ các đường mutation hoặc serialization bị trùng

## Workflow

- theo dõi area state trong [docs/features/feature-tables.md](../../feature-tables.md)
- giữ bằng chứng audit lặp lại trong `docs/audits/architecture-conformance/`
- theo dõi executable work trong `docs/tasks/`
- giữ folder này tập trung vào overview, risk, constraint, và link liên quan
